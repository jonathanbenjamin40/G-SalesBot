# תבנית סוכן אישי לסמנכ"ל — גינדי החזקות

זו תבנית בסיס לשכפול. כל סמנכ"ל מקבל **repository נפרד משלו** ב-GitHub, בנוי מהתבנית הזו.

## מבנה הקבצים (4 דברים בלבד)

```
index.html      — הכל בפנים: מבנה, עיצוב (CSS), ולוגיקה (JS), כולל בלוק ההגדרות
manifest.json   — הגדרות PWA (שם, אייקונים) — קובץ נפרד בגלל דרישת דפדפנים
sw.js           — Service Worker לתמיכה בהתקנה כאפליקציה — קובץ נפרד, לא לגעת
icons/          — כל האייקונים (מלוגו גינדי) — זהים לכל הסמנכ"לים, לא לגעת
```

## שכפול לסמנכ"ל חדש — 2 קבצים לערוך, שאר הכל נשאר כמו שהוא

### 1. `index.html`

בתוך התג `<script>` בתחתית הקובץ, יש בלוק מסומן בבירור:

```js
window.AGENT_CONFIG = {
  WEBHOOK_URL: "...",
  AUTH: { phone: "...", password: "..." },
  WELCOME_MESSAGE: "...",
  AGENT_NAME: "...",
};
```

**זה השדה היחיד שצריך לערוך.** אין צורך לגעת בשום דבר אחר בקובץ — לא ב-CSS ולא בשאר ה-JS.

בנוסף, יש כמה placeholders בראש הקובץ (בתוך `<title>`, `<meta>`, ובתוך גוף העמוד) שכתובים בפורמט `__VP_XXX__` — יש להחליף אותם (חיפוש-והחלפה גלובלי) בשם/תיאור המתאימים לסמנכ"ל:

| Placeholder | להחליף ב... | דוגמה |
|---|---|---|
| `__VP_TITLE__` | כותרת הדפדפן | הסוכן האישי של שרון \| גינדי החזקות |
| `__VP_DESCRIPTION__` | תיאור קצר | סוכן AI אישי לניתוח נתוני שירות עבור שרון, VP שירות |
| `__VP_SHORT_NAME__` | שם קצר (למסך הבית בנייד) | סוכן שרון |
| `__VP_SPLASH_CAPTION__` | כיתוב אנגלית במסך הפתיחה | Sharon's Robot |
| `__VP_LOGIN_TITLE__` | כותרת במסך ההתחברות | הסוכן האישי של שרון |
| `__VP_AGENT_NAME__` | שם בכותרת הצ'אט | הסוכן האישי של שרון |

### 2. `manifest.json`

אותו עיקרון — placeholders בפורמט `__VP_XXX__`:

| Placeholder | להחליף ב... |
|---|---|
| `__VP_FULL_NAME__` | שם מלא, למשל "הסוכן האישי של שרון" |
| `__VP_SHORT_NAME__` | שם קצר, למשל "סוכן שרון" |
| `__VP_DESCRIPTION__` | תיאור, למשל "סוכן AI אישי עבור שרון, VP שירות" |

## מה **לא** נוגעים בו בעת שכפול

- `sw.js` — זהה לכל הסוכנים, אין תלות בסמנכ"ל ספציפי.
- `icons/` — הלוגו של גינדי זהה לכולם.
- ה-CSS בתוך `index.html` — העיצוב (שחור-זהב-לבן) אחיד לכל הסמנכ"לים בשלב זה. שינויי גוון/עיצוב פרטניים לכל סמנכ"ל ייעשו בהמשך, בנפרד.

## פריסה ל-Vercel (לכל repo בנפרד)

1. Vercel → New Project → בחר את ה-repository של הסמנכ"ל הספציפי.
2. Deploy (ללא Build Command — אתר סטטי, Framework Preset: Other).
3. תקבל כתובת ייחודית (למשל `sharon-agent.vercel.app`).

- [ ] העתקתי את 4 הקבצים/תיקיות מהתבנית.
- [ ] עדכנתי את `WEBHOOK_URL` ל-Production URL של ה-workflow הספציפי של הסמנכ"ל הזה ב-n8n.

- [ ] עדכנתי `WELCOME_MESSAGE` ו-`AGENT_NAME`.

- [ ] פרסתי ל-Vercel ובדקתי שהצ'אט מתחבר בהצלחה ל-Webhook.
